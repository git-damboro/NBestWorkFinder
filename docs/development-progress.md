## 4. 已完成内容

| 分类 | 已完成项 |
|---|---|
| 后端认证 | 注册、登录、刷新 Token、退出接口已实现 |
| 前端认证 | 已新增认证 API 封装与 `AuthContext` 登录态管理 |
| 前端认证 | 请求层已支持自动注入 Bearer Token，并在 401 时清理本地登录态 |
| 前端认证 | 已新增登录页与注册页 |
| 前端认证 | 已接入 `AuthProvider` 并完成公开/受保护路由结构 |
| 前端认证 | 布局层已显示当前用户邮箱并支持退出登录 |
| 后端安全 | 已收紧业务接口匿名访问，业务页面请求必须携带登录 Token |
| 简历隔离 | `resume` 上传、列表、详情、删除、导出 PDF、重新分析均已接入 `@AuthenticationPrincipal Long userId` |
| 简历隔离 | 简历查询、删除与重分析均改为 `id + userId` 双条件，其他用户的简历对当前用户表现为不存在 |
| 简历隔离 | 简历去重从全局 `fileHash` 调整为当前用户内 `fileHash + userId` 去重 |
| 简历隔离 | 已增加启动时 PostgreSQL 轻量索引校准，避免旧库全局 `file_hash` 唯一索引阻止多用户上传相同简历 |
| 职位匹配安全 | `job` 简历匹配已改为校验当前用户自己的简历，避免通过 `resumeId` 引用他人简历 |
| JWT 体系 | AccessToken + RefreshToken 已实现，RefreshToken 已存入 Redis |
| 基础设施 | PostgreSQL、Redis、RustFS、Swagger、PDF 导出、Redis Stream 已可用 |
| 核心业务 | 简历分析、模拟面试、知识库 RAG、职位管理后端已具备 |
| 安全修复 | 已完成简历上传 CORS 预检修复，并补了相关测试 |
