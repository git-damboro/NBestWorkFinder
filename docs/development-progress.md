## 7. 技术债与风险

| 类型 | 内容 | 当前结论 |
|---|---|---|
| 历史数据 | 旧库中可能存在 `user_id=null` 的知识库或 RAG 会话记录 | 开发阶段可接受默认不可见，后续如需要可补迁移脚本 |
| 代码收口 | 仍有少量旧的无 `userId` 内部方法保留在 repository / service 层 | 当前外部入口已收口，后续可继续清理内部冗余接口 |
| 测试覆盖 | 当前以最小单测为主 | 适合当前开发节奏，后续可逐步补跨用户拒绝场景 |
| 全链路联调 | 职位前端尚未补完 | 现阶段主要影响产品完整性，不影响已完成模块的隔离正确性 |
