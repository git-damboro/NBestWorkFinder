# 投递辅助闭环手动联调清单

## 联调前准备

| 项 | 检查内容 | 通过标准 |
|---|---|---|
| 前端 | 本地系统页可打开 | `http://localhost:5173` 可正常访问 |
| 后端 | 接口可用 | `http://localhost:8080` 已启动，接口无 5xx |
| 扩展 | 已重新加载最新代码 | 浏览器扩展管理页已点击“重新加载” |
| 登录态 | 系统已登录一次 | 打开系统页面后，扩展可识别登录态 |
| BOSS 页面 | 有可访问的列表页与详情页 | 至少准备 1 个列表页和 1 个职位详情页 |

## 推荐联调顺序

| 顺序 | 场景 | 操作 | 期望结果 |
|---:|---|---|---|
| 1 | 扩展登录态同步 | 打开系统页并登录，再打开扩展 Popup | 状态显示“已连接”，不是“未登录” |
| 2 | BOSS 列表页同步 | 进入 BOSS 列表页，点击“同步当前页职位” | 最近任务成功；最近批次出现批次号、状态、条数 |
| 3 | 草稿页恢复 | 点击“打开职位草稿”，或系统页直接打开 `/jobs/drafts` | 能进入对应批次；无批次时显示空态，不应一直转圈 |
| 4 | 草稿页筛选 | 切换“未导入 / 已导入 / 待补全 JD / 已补全 JD / 高匹配” | 列表数量、统计数字、全选范围与筛选结果一致 |
| 5 | 单条来源跳转 | 草稿卡片点击“打开来源” | 新标签页打开原始职位链接 |
| 6 | BOSS 详情补全 | 在详情页点击“补全当前 JD” | 最近任务成功；草稿页出现完整 JD / 匹配摘要 / 开场话术 |
| 7 | 草稿单条重算 | 草稿卡片点击“重新计算匹配” | 当前卡片匹配分、摘要、话术刷新 |
| 8 | 批量导入工作台 | 选择 1 条或多条未导入草稿，点击“批量导入” | 成功提示正确；按钮数量与可导入数一致 |
| 9 | 工作台聚焦 | 当仅导入 1 条时点击“查看职位工作台” | 工作台自动聚焦到刚导入的职位 |
| 10 | 最近批次恢复 | 清掉扩展本地 `lastBatch` 后点“恢复最近批次” | 可重新拉回最近可恢复批次 |
| 11 | 标记已投递 | 在职位详情中点击“标记已投递” | 状态变为已投递，时间线出现状态变化记录 |
| 12 | 添加跟进记录 | 添加一条 BOSS 沟通记录并设置下一步跟进时间 | 时间线出现记录，职位详情显示下一步跟进时间 |
| 13 | 状态推进 | 将职位改为面试中 / Offer / 拒绝 | 每次状态变化都进入时间线 |

## 重点观察项

| 模块 | 观察点 | 正常表现 | 异常表现 |
|---|---|---|---|
| 草稿页 | 无最近批次 | 显示空态和引导文案 | 页面一直 loading |
| 草稿页 | 导入按钮 | 显示“可导入条数” | 已选很多但实际不可导入时仍可点 |
| 草稿页 | 恢复提示 | 仅在恢复成功后显示 | 恢复失败也显示“已恢复最近批次” |
| 扩展 Popup | 最近批次信息 | 有状态、条数、更新时间 | 只显示批次号，其他字段为空 |
| 扩展与 BOSS | 脚本注入 | 同步/补全可直接执行 | 报未注入脚本；刷新页面后应恢复 |
| 后端恢复逻辑 | 历史批次兼容 | 可恢复未完成且未过期，兼容旧数据 | 历史可恢复批次找不回 |
| 投递跟进 | 状态时间线 | 状态变化自动记录，手动记录可追加 | 只改状态但没有记录 |

## 常见异常与定位建议

| 现象 | 优先检查 | 定位建议 |
|---|---|---|
| 扩展显示未登录 | 系统页是否已登录、扩展是否重载 | 先打开系统页并刷新一次，再开 Popup |
| 点击同步/补全没反应 | 当前页是否真的是 BOSS 列表/详情页 | 看 Popup 状态文案是否识别成 `list` / `detail` |
| 报“未注入扩展脚本” | 页面是否在扩展更新前打开 | 直接刷新当前 BOSS 页面后重试 |
| 草稿页没有自动恢复 | 后端是否存在最近可恢复批次 | 先用扩展“恢复最近批次”，再看后端返回 |
| 详情补全找不到草稿 | 是否先做过列表页同步 | 补全依赖最近批次匹配，需先同步列表页 |
| 单条重算失败 | 草稿是否已有完整 JD | 没有完整 JD 时应先到详情页补全 |

## 联调通过标准

| 维度 | 通过条件 |
|---|---|
| 主链路 | `列表同步 → 草稿筛选 → 详情补全 → 匹配/话术展示 → 导入工作台` 至少完整跑通 1 次 |
| 恢复能力 | 扩展“恢复最近批次”和草稿页无 `batchId` 自动恢复都可工作 |
| 空态处理 | 无最近批次、无筛选结果、未注入脚本时都有明确提示 |
| 单条闭环 | 至少 1 条职位可从来源打开、补全 JD、重算匹配、导入工作台 |
| 批量闭环 | 至少 2 条职位批量导入成功，且不会重复导入已导入项 |
| 投递闭环 | 至少 1 条职位可标记已投递、添加沟通记录、设置下一步跟进、推进到面试中 |

## 联调时建议顺手记录

| 记录项 | 示例 |
|---|---|
| BOSS 页面 URL | 列表页和详情页各 1 个 |
| 批次号 | 如 `jdb_xxx` |
| 失败提示原文 | 直接复制弹窗/页面提示 |
| 浏览器行为 | 是否需要刷新、是否需重新打开 Popup |
| 数据结果 | 匹配分、摘要、话术是否刷新 |
