# NBWF 浏览器扩展 MVP

## 当前能力

- 在 `BOSS` 职位列表页同步当前页职位到系统的职位草稿池
- 在 `BOSS` 职位详情页补全完整 `JD`
- 自动读取本地系统页面的登录态，无需手动粘贴 `token`
- 同步完成后直接打开系统中的职位草稿页
- 支持恢复后端最近一次职位草稿批次
- 用户切换页面后，已点击的同步任务仍由扩展后台继续执行

## 本地加载方式

1. 打开 Chrome / Edge 的扩展管理页
2. 开启“开发者模式”
3. 选择“加载已解压的扩展程序”
4. 选择当前目录下的 `browser-extension`

## 使用前提

- 前端系统页面运行在 `http://localhost:5173` 或 `http://127.0.0.1:5173`
- 后端接口运行在 `http://localhost:8080` 或 `http://127.0.0.1:8080`
- 已在系统中完成登录

## 使用步骤

1. 先打开系统页面并登录一次，扩展会自动同步 `nbwf_auth_session`
2. 打开 `BOSS` 列表页，在扩展弹窗中选择简历
3. 点击“同步当前页职位”，生成职位草稿批次
4. 如需查看结果，点击“打开职位草稿”
5. 打开某个职位详情页后，点击“补全当前 JD”
6. 返回系统查看更完整的匹配分、摘要和开场话术
7. 如果本地批次缓存丢失，可点击“恢复最近批次”重新找回最近一次草稿批次

## 当前约束

- 当前仅支持 `localhost/127.0.0.1` 的本地开发地址
- 详情页补全依赖“先做过一次列表页同步”，用于定位对应的草稿项
- `BOSS` 页面 DOM 如后续调整，可能需要继续补充选择器
- “恢复最近批次”恢复的是当前登录用户在后端的最近草稿批次，不一定来自当前打开的 BOSS 页面

## 手动验证

1. 打开 `http://localhost:5173` 并完成登录
2. 在 BOSS 列表页打开扩展，确认“同步当前页职位”按钮可点击
3. 点击“同步当前页职位”，确认最近任务变为成功，且可打开职位草稿
4. 打开某个职位详情页，点击“补全当前 JD”，确认最近任务变为成功
5. 清空扩展本地 `lastBatch` 后重新打开扩展，点击“恢复最近批次”，确认最近批次重新出现
6. 在非 BOSS 页面打开扩展，确认同步按钮禁用且状态提示清晰
7. 如果弹出“当前页面未注入扩展脚本”，先刷新当前 BOSS 页面，再重新执行同步或补全
